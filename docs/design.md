# セカンドワンナイト人狼 デザインシステム

トップ LP([frontend/src/pages/index.tsx](../frontend/src/pages/index.tsx) / [frontend/src/styles/lp.module.scss](../frontend/src/styles/lp.module.scss))と werewolf 画面で使っているデザイン言語を体系化した文書。

**適用範囲**: トップ LP と、セカンドワンナイト人狼のゲーム画面(ルーム・進行画面)。他4ゲーム(timebomb / hideout / decrypt / fakeartist)は対象外。

**運用**: LP・werewolf 画面のデザインを変えたら同じ PR で本書も更新する(docs/architecture/ と同じ運用)。

---

## 1. デザインコンセプト

**「静かな village mystery」— 澄んだ朝の村と、疑心が満ちる夜の対比。**

- ホラー的な禍々しさではなく、**上質で落ち着いた和モダン**の空気で人狼の緊張感を表現する
- 世界観の軸は **朝(ライト面)と夜(ダーク面)** の2面構成。LP ではセクションごとに切り替えている
- 色に意味を持たせる:
  - **ティール(青緑)= 村・理性・朝・信頼**
  - **ローズ(赤桃)= 人狼・疑い・夜・血の気配**
  - この2色の「対比と混じり合い」(グラデーション・divider)が、正体を探り合うゲーム性のメタファー
- 余白と文字間(letter-spacing)を贅沢に使い、「間」で緊張感をつくる

## 2. カラーシステム

### 2.1 基本パレット

SCSS 変数名は lp.module.scss の定義に準拠。

| トークン | 値 | 役割 |
| --- | --- | --- |
| `$ink` | `#1e3b44` | 基調色。見出し・本文の主テキスト、プライマリボタン背景 |
| `$ink-deep` | `#17454f` | `$ink` の hover・強調 |
| `$teal` | `#35a8b4` | アクセント1(村・理性)。サブ見出し、ボーダー、リンク hover |
| `$teal-soft` | `#7fd0d6` | ダーク面でのティール(暗背景上で視認性を確保した明度) |
| `$rose` | `#e88f94` | アクセント2(人狼・疑い)。小さなアクセント(スクロールキュー等) |
| `$rose-soft` | `#e9a7be` | ダーク面でのローズ、第三陣営カラー |
| `$rose-deep` | `#c4646e` | ライト面でローズを文字に使う時(タグライン・エラー文言) |
| `$mist` | `#effdfe` | ライト面の基本背景 |
| `$paper` | `#fbfefe` | ライト面のセクション背景(mist より白寄り) |
| `$night` | `#142f37` | ダーク面の基本背景(役職紹介セクション等) |
| `$night-deep` | `#0d2229` | 最も深い背景(フッター) |
| `$text-sub` | `#43626b` | ライト面の補足テキスト・リード文 |
| `$label` | `#5a8189` | ラベル・キャプション・eyebrow |

### 2.2 陣営カラー

役職カード等でゲーム上の陣営を示す色。バッジ(枠+文字)や hover 時のボーダー・グローに使う。

| 陣営 | 色 | 備考 |
| --- | --- | --- |
| 人狼陣営 | `#f0949b`(ローズ系) | ダーク面用。`rgba(240, 148, 155, 0.55)` の枠線 |
| 村人陣営 | `$teal-soft` `#7fd0d6` | `rgba(127, 208, 214, 0.55)` の枠線 |
| 第三陣営 | `$rose-soft` `#e9a7be` | `rgba(233, 167, 190, 0.55)` の枠線 |

### 2.3 ライト面 / ダーク面の使い分け

| 面 | 背景 | 主テキスト | 補足テキスト | 用途 |
| --- | --- | --- | --- | --- |
| ライト | `$mist` / `$paper` | `$ink` | `$text-sub` / `$label` | 情報を読ませる場面。ルール説明・ロビー・昼フェーズなど |
| ダーク | `$night` / `$night-deep` | `#f2fbfb` | `rgba(242, 251, 251, 0.7)` | 世界観・緊張感を出す場面。役職紹介・夜フェーズなど |

werewolf の現行ゲーム画面では、ロビーはライト面、ゲーム中(turn 1〜3)は夜系ダーク面で連続させる。役職選択だけでなく、議論・投票も暗背景に置き、緊張感を保ったままフェーズ差分を色味で出す。

- ダーク面のカード: `rgba(255, 255, 255, 0.045)` 背景 + `rgba(255, 255, 255, 0.1)` ボーダー
- ダーク面のアクセントは soft 系(`$teal-soft` / `$rose-soft`)を使う。原色の `$teal` / `$rose` は暗背景では沈む

### 2.4 グラデーション

2色の対比を象徴する要素として限定的に使う。

- **divider**: `linear-gradient(90deg, $teal, $rose)`(ダーク面では soft 系)。幅 56px × 高さ 1px の細線
- **CTA 背景**: `linear-gradient(130deg, $ink-deep 0%, #2e7d8a 48%, #c97f92 100%)` — 夜から夕暮れへの遷移
- **セクション遷移**: `linear-gradient(180deg, $paper 0%, #f8eff0 65%, #f4e5e7 100%)` のように白→淡ローズで次のセクションへつなぐ
- **背景の radial-gradient**: `radial-gradient(120% 90% at 78% 30%, #f7eff2 0%, $mist 55%)` のような淡い光だまり

## 3. タイポグラフィ

### 3.1 フォントファミリー

Google Fonts で読み込み([frontend/src/pages/_document.tsx](../frontend/src/pages/_document.tsx))。

| 用途 | フォント | ウェイト |
| --- | --- | --- |
| 見出し・数字・情緒的な文言 | `'Shippori Mincho', serif` | 500 / 600 / 700(見出しは 600 基本) |
| 本文・UI | `'Zen Kaku Gothic New', sans-serif` | 400 / 500 / 700 |

- 明朝(Shippori Mincho)は「和の物語性・格調」を担う。セクションタイトル、役職名、大きな数字(ステップ番号・統計値)、タグラインに使う
- ゴシック(Zen Kaku Gothic New)は可読性を担う。リード文、説明文、ボタン、ラベルに使う
- ボタン・ラベル類は **font-weight: 700 + 広い letter-spacing** でゴシックを引き締める

### 3.2 サイズスケール

レスポンシブは `clamp()` で流動的に。固定ブレークポイント切り替えより優先する。

| 役割 | サイズ | フォント |
| --- | --- | --- |
| ヒーロータイトル(主) | `clamp(36px, 5.2vw, 70px)` | 明朝 600 |
| ヒーロータイトル(副) | `clamp(19px, 2.6vw, 30px)` | 明朝 600 |
| CTA タイトル | `clamp(30px, 5vw, 52px)` | 明朝 600 |
| セクションタイトル | `clamp(25px, 3.8vw, 40px)` | 明朝 600 |
| カード見出し(h3) | 20〜23px | 明朝 600 |
| 大きな数字(統計・ステップ) | 36〜38px / `clamp(28px, 3vw, 36px)` | 明朝 |
| リード文・本文 | `clamp(13.5px, 1.5vw, 15.5px)` | ゴシック 400 |
| ボタン | 14px | ゴシック 700 |
| 補足・キャプション | 12〜13.5px | ゴシック |
| eyebrow(セクションラベル) | 11px | ゴシック 700 |
| 最小ラベル(statLabel 等) | 9〜10.5px | ゴシック 700 |

### 3.3 letter-spacing / line-height

「間」をつくる最重要パラメータ。要素が小さいほど letter-spacing を広くとる。

| 要素 | letter-spacing |
| --- | --- |
| eyebrow・極小ラベル | `0.32em`〜`0.44em` |
| ヒーロー副タイトル | `0.42em` |
| タグライン・カード見出し | `0.18em`〜`0.22em` |
| ボタン | `0.18em` |
| セクションタイトル | `0.14em`〜`0.16em` |
| ヒーロー主タイトル | `0.1em` |
| 本文・キャプション | `0.12em`〜`0.16em` |

line-height: 本文・リード文は **1.9〜2.2**(ゆったり読ませる)。見出しは 1.15 前後。

## 4. 形状・エフェクト

### 4.1 角丸

| 要素 | border-radius |
| --- | --- |
| ボタン・チップ・バッジ | `999px`(完全な pill) |
| カード | `14px`〜`16px` |
| カード内画像・URL 表示枠 | `10px`〜`12px` |

### 4.2 ボタン

| 種別 | スタイル |
| --- | --- |
| Primary(ライト面) | 背景 `$ink`、文字 `#f2fbfb`。hover で `$ink-deep` + 浮き上がり |
| Ghost(ライト面) | 透明背景 + `rgba(30, 59, 68, 0.35)` 枠。hover でティールの淡い背景 `rgba(53, 168, 180, 0.1)` |
| Primary(ダーク面) | 背景 `#fdfefe`、文字 `$ink-deep`(反転) |
| Ghost(ダーク面) | `rgba(255, 255, 255, 0.55)` 枠、白文字。hover で `rgba(255, 255, 255, 0.14)` 背景 |

共通: `min-height: 50px`、`padding: 12px 34px`、hover で `translateY(-3px)` + 大きめの柔らかい影。

### 4.3 影・ボーダー

- 影は **大きくぼかして薄く**: `0 20px 40px rgba(23, 69, 79, 0.1)` 程度。hover 時のみ付与が基本
- 影の色は黒ではなく `$ink` 系の色味(`rgba(23, 69, 79, …)` / `rgba(30, 59, 68, …)`)を使う
- ボーダーは 1px 固定。ライト面はティール系の淡い透過(`rgba(53, 168, 180, 0.18)`)、ダーク面は白の透過(`rgba(255, 255, 255, 0.1)`)

### 4.4 ガラス表現・グロー

- 半透明カード: `rgba(255, 255, 255, 0.8)` + `backdrop-filter: blur(6px)`(ライト面のグラデーション背景上)
- グロー(光だまり): `radial-gradient(circle, rgba(127, 208, 214, 0.16), transparent 70%)` を大きな円で配置し `blur` + ゆっくり明滅。ダーク面の装飾に使う
- 装飾リング: 1px の透過ボーダー円をゆっくり脈動させる(`ringPulse`)

## 5. モーション

### 5.1 基準値

- **共通イージング**: `cubic-bezier(0.22, 0.61, 0.36, 1)`(SCSS 変数 `$ease`)。UI の transition は原則これ
- UI の transition: **0.3〜0.4s**(hover 等)
- 出現アニメーション: **0.9〜1.4s**(reveal / fadeUp)
- 環境アニメーション(グロー・浮遊): **8〜14s** の ease-in-out 無限ループ

### 5.2 パターン

| 名前 | 内容 | 用途 |
| --- | --- | --- |
| `reveal` / `fadeUp` | opacity 0 + `translateY(24〜30px)` → 定位置 | スクロール出現・初期表示。delay を 0.15s 刻みで順次 |
| `charIn` / `charInTeal` | 1文字ずつ blur + 浮き上がりで出現。途中でローズ→定着色に変化 | ヒーロータイトルの演出 |
| `heroFloat` | ±11px 程度のゆっくりした上下浮遊 | メインビジュアル |
| `breatheGlow` | scale 1→1.07 + opacity の明滅 | グロー装飾 |
| `ringPulse` | scale 1→1.035 の脈動 | 装飾リング |
| hover 浮遊 | `translateY(-3〜-5px)` + 影 | ボタン・カード共通の hover |

### 5.3 原則

- 動きは「ゆっくり・静か」。バウンスや高速な動きは世界観に合わないので使わない
- **`prefers-reduced-motion: reduce` 対応は必須**。全 animation / transition を無効化し、出現系は最終状態(opacity 1)で表示する

## 6. 画像・イメージ

- **トーン**: メインビジュアル(`/images/hero.jpg`)は淡い色調・柔らかい光のイラスト。写真ではなくイラストで統一する
- **なじませ方**: 矩形のまま置かず、`mask-image: radial-gradient(...)` で輪郭を背景に溶かす。グロー・リングを重ねて浮遊させる
- **役職カード画像**(`/images/werewolf/roll/{n}.jpg`): 正方形(`aspect-ratio: 1/1`)+ `object-fit: cover` + 角丸 12px で統一
- **装飾モチーフ**: 舞い散る葉(`border-radius: 0 100% 0 100%` の葉形 + `leafFall`)。ティール/ローズの淡色で世界観を補強

## 7. ゲーム画面への適用ガイド

werewolf のルーム・進行画面で使う指針。

### 7.1 フェーズと面

| フェーズ | 面 | 備考 |
| --- | --- | --- |
| ロビー・ルーム作成 | ライト面(`$mist` / `$paper`) | 情報が読みやすいことを優先 |
| 役職選択(turn 1) | ダーク面(`$night`) | グロー装飾で夜の始まりを作る |
| 議論(turn 2) | 夜系ダーク面 | 役職選択より少し明るい藍。gold の灯で会話の温度を足す |
| 投票(turn 3) | 夜系ダーク面 | deep 藍 + rose ティントで緊張感を強める |
| 結果発表(turn 4) | 勝利陣営の色をアクセントに | 人狼勝利=ローズ系、村人勝利=ティール系、第三陣営=ローズソフト系 |

### 7.2 コンポーネント指針

- **ボタン**: LP の `.btnPrimary` / `.btnGhost` のスタイルを踏襲(pill・letter-spacing 0.18em・hover 浮遊)。ダーク面では反転版を使う
- **プレイヤー・役職カード**: 角丸 14〜16px、1px 透過ボーダー、hover で `-5px` 浮遊。陣営が判明している場合は陣営カラーの枠色
- **バッジ・チップ**(陣営表示・ステータス): pill 型、`font-size: 10〜12px`、`letter-spacing: 0.12〜0.22em`、陣営カラーの枠+文字
- **見出し**: フェーズ名・役職名は明朝(Shippori Mincho 600)+ 広い letter-spacing。eyebrow(小ラベル)+ タイトルの2段構成を踏襲
- **エラー・警告**: ライト面 `$rose-deep`、ダーク面 `#ffd7da`
- **モーション**: フェーズ切替は fadeUp 系のゆっくりした遷移。reduced-motion 対応を忘れない
- **待機/招待**: `EntryCard` / `InvitePanel` は中央カードとコンパクトな招待カードで構成する。roomCode は待機中/終了後だけ目立たせる
- **背景**: `PhaseBackground` で turn / 勝利陣営に応じて、ロビー(day)、役職選択(night)、議論(discussion)、投票(voting)、結果(勝利陣営色)を切り替える
- **花びら演出**: `SakuraParticles` は待機中は `ambient`、勝利演出では `celebration` を使う。勝利時は `victoryPalette` で陣営色を指定する
- **勝利演出**: `VictoryOverlay` で「勝利演出 → 結果表示 → ロビー復帰」を全画面 overlay として出す。結果表は overlay 内に収め、招待カードと重ねない

### 7.3 実装メモ

- デザイントークンは [tokens.scss](../frontend/src/styles/tokens.scss) を正とする。LP と werewolf 画面の SCSS modules は `@use` で参照し、色・フォント・easing を共有する
- フォントは _document.tsx で全ページ読み込み済みのため、ゲーム画面側で追加の読み込みは不要
