# werewolf 待機画面・終了画面リデザイン設計書

セカンドワンナイト人狼の待機画面(turn 0 / 4 のロビー)と終了画面(勝利演出・結果表示)を、ゲーム中画面と同じ「夜の和モダン」世界観でゼロベース再設計する。

## 背景・課題

ゲーム中画面(役職選択・議論・投票)は夜系トークンで統一済みだが、待機・終了は旧トーンのまま。具体的な課題:

1. 待機画面が単調・退屈で、開始前のワクワク感がない
2. 設定 UI(役職プリセット・役職カスタム・制限時間)がフッターに雑然と並び分かりにくい
3. 終了画面で背景色と時間設定 UI の色が被って見にくい
4. 勝利演出中に死亡者(処刑・銃撃された人)が誰か分からない
5. 終了画面の盛り上がり(種明かしのドラマ)が不足

## コンセプト

**一日の時間軸で3画面をつなぐ**: 待機=夕暮れ(dusk)→ ゲーム中=夜(既存)→ 終了=夜明け(dawn)。

- 対象デバイス: スマホ・PC 両方同等(スマホ=1カラム縦積み、PC=中央 720〜960px カラム+余白に遠景演出)
- 通信契約・backend・reducer の状態遷移は**変更しない**。演出はすべて表示層のローカル state で制御する

## 待機画面「宵の口の集会場」

### シーン

- `PhaseBackground` に `dusk` シーンを追加。茜色→藍色の縦グラデーション+遠景の山並みシルエット+一番星
- turn 0 → turn 1 で「日が沈む」トランジションにより既存の夜背景へ接続
- `SakuraParticles` に夕暮れ用の色味 mode を追加して継続使用

### 画面構成(上から順)

1. **ヘッダー**: ゲームロゴ(小)+ルームコード招待カード(`InvitePanel` 改修。筆文字風の大きめコード表示+コピー/共有)+「遊び方」リンク
2. **提灯列(`LanternRow`・新規)**: 最大10個の提灯が並び、入室ごとに1つ灯る。人数プログレスを兼ね、開始可能人数(3人)に達すると提灯列が揺れて合図
3. **プレイヤーグリッド**: `UserField` の待機用レイアウト。入室時に「のれんをくぐる」登場アニメ。自分のアイコンタップでアイコン変更(既存 `IconPicker`)、他人のアイコンタップで「つつく」小演出(ローカルのみ・通信なし)
4. **お品書きパネル(`MenuPanel`・新規)**: `WerewolfSet` / `RollCustomize` / `LimitTimeSelector` を1枚のカードに統合。上段=プリセット選択タブ、中段=役職と人数(現在の構成が常に見える)、下段=議論時間。役職合計と参加人数の過不足を「あと1役職足りません」等で明示
5. **フッター**: GAME START(主役・開始条件未達なら理由つき無効表示)/ 退出

## 終了画面「夜明けの種明かし」

### シーン

- `PhaseBackground` に `dawn` シーンを追加。演出進行に合わせ「深夜 → 白み始め → 朝焼け」と明けていく。勝利陣営色は朝焼けに混ぜ、陣営色ベタ塗りをやめる
- 演出を閉じた後の turn 4 ロビーは dawn パレットで統一。UI 側も dawn トークンで追従し、**色被り課題を解消**

### 勝利演出の3幕構成(`VictoryOverlay` を再設計)

**第1幕: 種明かし(`RoleRevealAct`)**

- 全プレイヤーカードが並び、1枚ずつ順に裏返して役職開示(人狼系は最後に開示してタメを作る)
- 死亡者はカードに `DeadMarker`(モノクロ化+斜めの「散」帯+線香の煙エフェクト)を付け、**以降の全幕で維持**
- 各カードの下に投票先を小さく矢印表示

**第2幕: 勝敗発表(`VerdictAct`)**

- 背景が夜明けへ転換。勝利陣営バナー(筆文字系タイポを1文字ずつ)+勝者カードが前に出て発光、敗者は一歩引く
- 陣営に応じた紙吹雪/花びら(既存 celebration 資産を拡張)

**第3幕: 結果とロビー復帰(`ResultScroll`)**

- 巻物風の結果一覧: 役職・生死・投票先・勝敗を1行で表示(旧 `result.tsx` を置換)。死亡者マーカー継続
- 「もう一度遊ぶ(GAME START)」「ロビーに戻る」を明示。閉じると dawn パレットの待機画面へ

### 操作

- 幕送りはタップ/クリック(自動送り+スキップ可)。演出の全員同期は取らずローカル進行(現行踏襲)

## 部品一覧

### 新規

| 部品 | 内容 |
| --- | --- |
| `LanternRow` | 提灯列。人数プログレス+開始可能の合図 |
| `MenuPanel` | プリセット/役職構成/議論時間の統合設定パネル。開始条件の過不足表示を内包 |
| `RoleRevealAct` | 第1幕: 役職順次開示+死亡者マーカー+投票矢印 |
| `VerdictAct` | 第2幕: 夜明け転換+陣営バナー+勝者前出し |
| `ResultScroll` | 第3幕: 巻物風結果一覧 |
| `DeadMarker` | 死亡者表示。全幕・結果で共用 |

### 改修

| 部品 | 改修内容 |
| --- | --- |
| `PhaseBackground` | `dusk` / `dawn` シーン追加、シーン間トランジション |

### 統合されて消える部品

- `WerewolfSet` / `RollCustomize` / `LimitTimeSelector` → `MenuPanel` に統合(ロジックは流用)
- `result.tsx` → `ResultScroll`

## 技術方針

- 通信契約・reducer の状態遷移は不変。演出は表示層のローカル state(現行 `VictoryOverlay` と同方式)
- 死亡者判定は既存 state の `roll.punishmentFlg`(処刑=最多得票、暗殺者・独裁者の銃撃時に backend が立てる)から純粋関数で導出し、`victory.ts` 側に置いてユニットテスト対象にする
- アニメーションは CSS 主体(トークン準拠)、`prefers-reduced-motion` 対応
- ブレークポイントはトークンで共通化
- テスト: reducer 変更なしのため既存テスト維持。新規純粋関数(死亡者導出・開示順ソート)にユニットテストを追加
